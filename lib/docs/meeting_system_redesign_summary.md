# 会议纪要系统重新设计总结

## 📋 概述

本次重新设计完全重构了会议纪要的主要功能，实现了一个清晰的三步骤流程，使用WebSocket和HTTP对应的库，提升了用户体验和系统稳定性。

## 🎯 新的用户流程设计

### **步骤1：会议设置页面**
- **功能**: 收集会议基本信息，选择模型配置
- **页面**: `MeetingSetupScreen`
- **特性**:
  - 会议标题、描述、参会人员设置
  - ASR模型和总结模型选择
  - 实时进度指示器
  - 配置验证和错误提示

### **步骤2：录音转录页面**  
- **功能**: 实时录音和转录
- **页面**: `MeetingRecordingScreen`
- **特性**:
  - 实时录音控制（开始/停止）
  - 实时转录显示
  - 智能音频数据累积处理
  - 转录进度显示

### **步骤3：总结问答页面**
- **功能**: 生成会议纪要和智能问答
- **页面**: `MeetingSummaryScreen`
- **特性**:
  - 基于原文生成结构化会议纪要
  - 智能问答功能
  - 实时聊天界面
  - 会议结果保存

## 🏗️ 架构设计

### **数据模型**
- `MeetingSession`: 会议会话状态管理
- `MeetingSessionStep`: 三步骤状态枚举
- `ChatMessage`: 问答消息结构

### **服务库**
- `ASRLibrary`: 语音识别服务抽象
  - `HTTPASRLibrary`: HTTP实现
  - `WebSocketASRLibrary`: WebSocket实现
- `SummaryLibrary`: 会议总结服务抽象
  - `HTTPSummaryLibrary`: HTTP实现
  - `WebSocketSummaryLibrary`: WebSocket实现

### **状态管理**
- `MeetingSessionProvider`: 会议会话状态管理
- 流程控制和进度跟踪
- 数据传递和持久化

## 🔧 技术实现

### **WebSocket vs HTTP支持**
1. **ASR服务**:
   - HTTP: 一次性转录和流式处理
   - WebSocket: 实时双向流式通信

2. **总结服务**:
   - HTTP: 批量生成和问答
   - WebSocket: 实时问答交互

### **关键特性**
1. **智能路由**: 根据模型支持自动选择实现方式
2. **错误处理**: 完善的异常处理和用户提示
3. **进度跟踪**: 可视化的流程进度显示
4. **数据传递**: 会话状态在页面间正确传递

## 📱 用户界面

### **主要改进**
1. **清晰的导航**: 三步骤进度指示器
2. **实时反馈**: 录音状态、转录进度、生成状态
3. **智能问答**: 聊天式问答界面
4. **响应式设计**: 支持移动端和桌面端

### **用户体验**
1. **引导式流程**: 从设置到总结的完整指导
2. **状态可视化**: 每个步骤的清晰状态显示
3. **错误友好**: 详细的错误信息和解决建议
4. **数据持久**: 会议结果自动保存

## 🚀 使用方法

### **开始新会议**
1. 点击首页的"+"按钮
2. 填写会议信息和选择模型
3. 开始录音并查看实时转录
4. 生成纪要和进行问答
5. 保存或分享会议结果
